## Codebase Patterns
- **CORRECTED**: Solution file is `Brighter.slnx` (modern XML format, NOT legacy .sln) - add projects as `<Project Path="..." />`
- **CORRECTED**: Repo DOES use central package management (`Directory.Packages.props` with `ManagePackageVersionsCentrally=true`) - do NOT put version attributes in csproj files
- **CORRECTED**: Source projects use `$(BrighterTargetFrameworks)` = `netstandard2.0;net8.0;net9.0;net10.0` (variable from `src/Directory.Build.props`)
- **CORRECTED**: Test projects use `$(BrighterTestTargetFrameworks)` = `net9.0;net10.0` (variable from `tests/Directory.Build.props`)
- **CORRECTED**: LangVersion is `latest` and Nullable is `enable` from Directory.Build.props
- **CORRECTED**: Version is 10.0.0.0 (not 9.0.0.0), MinVer 7.0.0 and SourceLink 10.0.103 via GlobalPackageReference
- `IAmA*` naming convention for role-based interfaces
- DI extensions follow `Use*` pattern on `IBrighterBuilder` with `Action<Options>` parameter; use `TryAddSingleton` for defaults
- `IServiceActivatorOptions.Subscriptions` provides registered subscriptions when `AddServiceActivator` has been called
- `IAmAProducerRegistry` is registered by `UseExternalBus` but only exposes `IAmAMessageProducer` instances (not `Publication` objects or topic keys)
- `Publication` class has NO `DataType` property - only Topic, MakeChannels, MaxOutStandingMessages, etc.
- All .cs files in the repo have the standard MIT licence header - new files MUST include it with 2025 copyright
- BDD test naming: `When_<action>_<subject>` for test class names, `It_Should_<expected>` for test methods
- NJsonSchema `JsonSchema.FromType()` generates PascalCase property names by default - match C# property casing in assertions
- `Subscription.DataType` is read-only `{ get; }` - to test null DataType, pass null with explicit name/channelName/routingKey to avoid NRE
- When testing subscription/publication generation in isolation, set `DisableAssemblyScanning = true` on options to prevent test types with `[PublicationTopic]` from being picked up
- `ReflectionTypeLoadException.Types` can contain null entries - always filter with `.Where(t => t != null)` before processing
- To test DI extensions on `IBrighterBuilder`, fake it with FakeItEasy: `A.CallTo(() => builder.Services).Returns(new ServiceCollection())` - no need for full `AddBrighter()` setup
- `dotnet build/test/restore` requires `dangerouslyDisableSandbox: true` due to sandbox file write restrictions
- This machine does NOT have net9.0 runtime - test runs on net9.0 will abort; net10.0 works
- `IAmConsumerOptions` (core Brighter) provides `Subscriptions` - use this instead of depending on ServiceActivator for subscription access
- In AsyncAPI generator code, prefer `TryAdd`/`TryGetValue` over `ContainsKey` + indexer to avoid double dictionary lookups and keep dedup logic idiomatic
- Strong-name signed assemblies make `InternalsVisibleTo` via csproj `<InternalsVisibleTo>` items fail without full public key - avoid where possible
- `PublicationTopicAttribute` exists in BOTH `Paramore.Brighter` and `Paramore.Brighter.AsyncAPI` - use fully qualified `Paramore.Brighter.AsyncAPI.PublicationTopic` in attribute usage to avoid CS0104 ambiguity
- All async APIs use `Task<T>` return types with `CancellationToken ct = default` parameter; FakeItEasy mocks need `A<CancellationToken>.Ignored` in call setups
- `#if NET8_0_OR_GREATER` conditional for `File.WriteAllTextAsync` since it's not available on netstandard2.0
