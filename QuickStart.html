Brighter

Getting Started

Brighter is a library and not a framework, so you can use it with

  • ASP.NET MVC or WebAPI,
  • Nancy,
  • OpenRasta,
  • Topshelf, and
  • others.

Brighter's objective is to decouple the Invoker and Receiver by a Command and without coupling the Invoker to a concrete Command. In addition, it can support orthogonal requirements such as

  • Retry,
  • Timeout,
  • Circuit Breaker, and
  • Logging

without polluting domain code.

Finally, Brighter can defer the work where synchronous execution is not required by using a task queue. The Broker Pattern architecture allows the tasks to be decoupled and distributed seamlessly to the Invoker.

Polly

Conflict Last Option

If Assembly Redirection does not resolve the issue then try:

  1. Fork the code,
  2. Rebuild with the required version,
  3. Fix any issues, and
  4. Deploy as a library and not a package.

The Brighter Core Library

The Brighter NuGet packages can be found using "Brighter" and the following packages are available:

  • Brighter.CommandProcessor - the core library, provides a Command Dispatcher and Command Processor
  • Brighter.ServiceActivator - provides consumer support for deferring long-running work to a task/work queue

The core library provides support for the producer, including but not limited to:

  • Brighter.MessagingGateway.RMQ - to transmit tasks to the broker for distribution
  • Brighter.MessageStore.RavenDB - to replay messages sent to the broker
  • Brighter.MessageStore.MSSQL - to replay messages sent to the broker
